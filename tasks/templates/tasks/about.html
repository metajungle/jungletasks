About

This is a simple Web application for managing tasks built by Jakob Henriksson.

The source code can be found on GitHub.

Frequently Asked Questions*

What's the point of this site?

The site provides a very simple way of keeping track of tasks.

Seriously, what's the point?

Ah, you mean, what's the point? In the sense that there are probably a thousand other web applications or desktop applications that do a better job?

Well, in that case there is no point. You should use whatever system you find useful for your purposes. If the best way for you keep track of what you need to get done is to keep a simple text file on your computer with your tasks, you should probably do that. If you already have an application that you are happy with that helps you get things done, you should probably stick with that.

What is a good workflow for managing tasks?

Whatever works for you.

Personally I like to organize everything around labels. Labels are just bins that you can put your tasks in. I typically have a general label for each project, let's call one such label Project. I put everything that has to do that particular project there. Then I keep other labels, e.g. Issues (or Bugs), Todo Next, RC (for release candidate), Showstoppers. I then organize all my tasks around these labels. I can move tasks around as I see fit. But everything is organized around these labels, but most importantly, their names is what keeps things moving forward.

Isn't this site too simple?

You mean it lacks a lot of features that are available in most other issue trackers? Yes, it is true. However, I have found many issue trackers just overly complex. I don't need most of that stuff.

What are some things I should be aware of when using the site?

Labels are first-class entities and not simply text strings that are associated with tasks. This means that if you delete a label, that label is gone. Yes, from all the tasks that previously had that label. Same thing if you rename a label, that old name is gone.

Does it have to be this way? No, but that's the way it has been implemented. The tasks you have completed is of some interests once you have completed them. That's why completed tasks are still viewable. They are also included if you export tasks. However, the labels are not stored in this way. They are just meant to help you organize your current tasks so that you know what needs to get done. They are your current window into your tasks. As you move forward that window will change (you will create/delete/rename labels). You old 'view' will not be saved. How you viewed things before is of less interest now. Your current label setup should be optimized to move things forward and get things done, now.

How do I mark a task as completed?

You simply click on the task text. One click marks it as completed, another click returns it to being not completed.

How do I delete a task?

You edit the task and delete the task from there.

How do I mark a task as high priority?

You edit the task and select the priority level from there.

Can I export my tasks so I can use them in other systems?

Yes. In any view that has a task you can click 'export' from the main menu. You will see a preview of a few tasks that shows how the exported tasks will be formatted. The actual export will contain all tasks in the view you were in when you clicked 'export'. If you want to export all tasks, click the "All" label first and then 'export' in the top-right corner. After previewing you can then export your tasks. A file will be saved to your local disk with all the tasks.

The format of the export is Org-Mode. It's basically a text file so you can do whatever you want with it.
